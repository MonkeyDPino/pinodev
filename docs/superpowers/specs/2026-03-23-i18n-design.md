## Overview

Add English/Spanish language switching to the pinodev portfolio site. The selected language persists across visits via `localStorage`. The switcher lives in the header as a pill toggle.

## Packages

Install three packages:

```
i18next
react-i18next
i18next-browser-languagedetector
```

`i18next-browser-languagedetector` handles `localStorage` persistence and initial browser-language detection automatically â€” no manual storage code needed.

## File Structure

```
src/
  i18n.ts                          # i18next initialization (imported once in main.tsx)
  locales/
    en.json                        # English translations
    es.json                        # Spanish translations
  components/
    LanguageSwitcher/
      LanguageSwitcher.tsx
      LanguageSwitcher.scss
```

## i18n Configuration (`src/i18n.ts`)

- Uses `LanguageDetector` plugin with detection order: `['localStorage', 'navigator']`, caching to `localStorage`
- Fallback language: `en`
- Resources: both locale files imported statically (no lazy loading needed for 2 languages)
- No namespaces â€” single flat `translation` namespace

## Language Switcher Component

**`LanguageSwitcher.tsx`**
- Reads `i18n.language` to determine the active language
- Calls `i18n.changeLanguage('es' | 'en')` on click
- Renders a two-segment pill: `[ES] [EN]` â€” active segment has teal background (`$secondary-color`) and dark text; inactive segment is gray text, transparent background

**Placement in `Header.tsx`**
- Added inside `.header__nav` (desktop) â€” right of the nav links
- Added inside `.header__nav_responsive` (mobile menu) â€” so mobile users can also switch language

## Components Updated

Every component that contains hardcoded user-visible text switches to `const { t } = useTranslation()` and `t('key')`:

| Component | Keys used |
|---|---|
| `Header` | `nav_home`, `nav_experience`, `nav_projects`, `nav_about` |
| `Home` | `home_role`, `home_description`, `home_cv_button` |
| `Experience` | `experience_title`, `experience_N_role`, `experience_N_description` |
| `Projects` | `projects_title`, `projects_0_description` |
| `AboutMe` | `about_title`, `about_p1`, `about_p2`, `about_p3` |
| `Certifications` | `certifications_title`, `certifications_N_description` |
| `Technologies` | `technologies_title` |

## Persistence

`i18next-browser-languagedetector` saves the selected language to `localStorage` under the key `i18nextLng`. On next visit the detector reads this key first before falling back to browser language or `en`.

## What Is NOT Translated

- Technology names (React, Node.js, Docker, etc.)
- Company names (Agentemotor, Universidad TecnolĂłgica de Pereira, etc.)
- Certification names
- Dates (e.g. "2022 - Present")
- Project names (e.g. "Giphy Piece")
- Alt text on SVG icons
