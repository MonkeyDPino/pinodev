**Date:** 2026-03-23
**Status:** Approved

## Overview

Add English/Spanish language switching to the pinodev portfolio site. The selected language persists across visits via `localStorage`. The switcher lives in the header as a pill toggle.

## Packages

Install three packages:

```
i18next
react-i18next
i18next-browser-languagedetector
```

`i18next-browser-languagedetector` handles `localStorage` persistence and initial browser-language detection automatically â€” no manual storage code needed.

## File Structure

```
src/
  i18n.ts                          # i18next initialization (imported once in main.tsx)
  locales/
    en.json                        # English translations
    es.json                        # Spanish translations
  components/
    LanguageSwitcher/
      LanguageSwitcher.tsx
      LanguageSwitcher.scss
```

## i18n Configuration (`src/i18n.ts`)

- Uses `LanguageDetector` plugin with detection order: `['localStorage', 'navigator']`, caching to `localStorage`
- Fallback language: `en`
- Resources: both locale files imported statically (no lazy loading needed for 2 languages)
- No namespaces â€” single flat `translation` namespace

### Rich text keys

Two places require the `Trans` component from `react-i18next` instead of plain `t()` calls, because they contain inline HTML markup:

**Home description** â€” contains a `<b>` segment and a colored `<span>`. Use three adjacent keys (`home_description_bold`, `home_description_colored`, `home_description_plain`) rendered as inline elements preserving the original `<b>` and `<span className="third-color-dark">` wrappers.

**AboutMe paragraphs 1 and 3** â€” contain `<strong>` inline segments. Split into prefix/bold/suffix keys (`about_p1_pre`, `about_p1_bold`, `about_p1_post` and `about_p3_pre`, `about_p3_bold`, `about_p3_post`) to preserve the `<strong>` wrappers.

**Note:** The source code contains a typo `"Enginner"` in the Home description colored segment. The translation files use the corrected spelling `"Engineer"`.

## Implementation Patterns

### Nav links â€” `Header` + `HeaderSection`

Call `t()` in `Header` and pass the translated string as the `title` prop to `HeaderSection`. `HeaderSection` requires no changes.

```tsx
// Header.tsx
<HeaderSection title={t('nav_home')} hash="home" />
```

### Data-array components

Three components (`Experience`, `Projects`, `Certifications`) define typed data arrays with translatable strings as literal values. Move the array definition inside the component body so `t()` can be called at render time:

```tsx
// Example â€” Experience.tsx
export default function Experience() {
  const { t } = useTranslation();
  const events = [
    { role: t('experience_0_role'), description: t('experience_0_description'), ... },
    { role: t('experience_1_role'), description: t('experience_1_description'), ... },
  ];
  // ...
}
```

Apply the same pattern to `Projects` and `Certifications`.

### Technologies categories

The `technologies` array entries keep their English `name` field unchanged (it drives the `nameColor` map). Translate the displayed category heading with a dynamic key derived from the name:

```tsx
<div className="technology__name" style={{ color: technology.nameColor }}>
  {t(`technologies_category_${technology.name.toLowerCase()}`)}
</div>
```

This requires the three keys `technologies_category_frontend`, `technologies_category_backend`, `technologies_category_tools` to exist in both locale files (keys are lowercase; source array has `"Frontend"`, `"Backend"`, `"Tools"`).

## Language Switcher Component

**`LanguageSwitcher.tsx`**
- Reads `i18n.language` to determine the active language
- Calls `i18n.changeLanguage('es' | 'en')` on click
- Renders a two-segment pill: `[ES] [EN]` â€” active segment has teal background (`$secondary-color`) and dark text; inactive segment is gray text, transparent background

**Placement in `Header.tsx`**
- Added inside `.header__nav` (desktop) â€” right of the nav links
- Added inside `.header__nav_responsive` (mobile menu) â€” so mobile users can also switch language

**Note:** The header nav contains four links only: Home, Experience, Projects, About me. The Certifications and Technologies sections exist on the page but do not have nav links.

## Persistence

`i18next-browser-languagedetector` saves the selected language to `localStorage` under the key `i18nextLng`. On next visit the detector reads this key first before falling back to browser language or `en`.

## What Is NOT Translated

- Technology names (React, Node.js, Docker, etc.)
- Company names (Agentemotor, Universidad TecnolĂłgica de Pereira, etc.)
- Certification names (treated as proper titles)
- Dates (e.g. "2022 - Present")
- Project names (e.g. "Giphy Piece")
- Icon and image alt attributes (LinkedIn, GitHub, Instagram, Profile image)
- Brand nickname ("pinodev") in Profile and Footer
- Owner's name ("Juan Esteban Pino") â€” proper noun, intentionally fixed across languages
- Copyright string in Footer (`"Copyright Â© Juan Pino 2025"`) â€” copyright notices are conventionally not localized
